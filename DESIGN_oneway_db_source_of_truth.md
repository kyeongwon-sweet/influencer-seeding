# 설계안: 안전한 양방향 동기화 (시트 입력 유지 + 구조적 오염 차단)

작성: Claude, 2026-07-14. 사용자 결정 = **양방향 유지**(시트 입력 유지). **실행 전 Codex+시트세션 합의 필요.**
(더 강한 대안: 단방향 DB 단일정본 = import 폐지. 워크플로 변경 부담 커서 이번엔 채택 안 함. 본 문서 맨 아래 참고.)

## 목표
시트↔DB 양방향을 유지하되, **"매일 왕복하며 오염이 번지고 compound되는" 구조적 원인을 제거**한다.

## 근본원인 3개 → 각 대응
| # | 원인 | 대응 |
|---|---|---|
| ① | **중복/겹친 날짜 열** — 같은 날짜 열이 여러 개라 sync가 엉뚱한 열에 씀 | 날짜당 열 **딱 1개·순서 고정**(정규화). export가 새 열 만들 때 중복 생성 금지 가드 |
| ② | **위치 기반 매칭**(행=위치, 열=위치) — 한 칸 밀리면 남의 값이 됨 | import/export를 **URL(행 키)+정규 날짜(열 키)로만** 매칭. 애매(중복열·미매칭)하면 **쓰지 말고 건너뛰고 알림**(추측 금지) |
| ③ | **매일 왕복 재확산** — 잘못 앉은 값이 import↔export로 번짐 | 복사-가드(이미 있음)+**수집 실측 대비 급변 알림**+**주1회 전수 스캔**으로 새 오염 조기 차단 |

→ ①②를 없애면 **번짐·compound 정지**. ③으로 신규 오염 조기 감지.

## 변경 항목 (파일·담당)
| 파일/작업 | 변경 | 담당 |
|---|---|---|
| 연동 시트 날짜 열 구조 | **중복 날짜 열 제거, 날짜당 1개·순서 고정**(1회). 이후 export가 중복 안 만들게 | **시트세션** |
| `Combined_Sheet_AppsScript.gs` (export/import) | 열 찾기를 **정규 날짜 헤더 1개로만**, 중복 헤더 감지 시 중단+경고. import는 URL 행 매칭 확정 | **시트세션** |
| `web/app/api/sponsored-posts/stats-import/route.ts` | 들어온 값 **URL+날짜 키 검증**, 애매하면 skip. 복사-가드 유지 + **수집 실측 대비 N배 급변 시 보류+알림**(alert-only, 절대규칙: 자동보정 금지) | **Claude(web)** |

## 마이그레이션 순서 (안전)
1. **DB 1회 정리** — 오염 게시물별 행 삭제 → 실측 최종값(재수집) 1행. 키 기반이라 안전. 백업 필수. **Claude**.
2. **시트 날짜영역을 DB에서 재생성** — 시트 date열 클리어 → export로 **정규 단일 열** 재구성(엉킨 이력 통째 교체). 백업 필수. **시트세션**.
3. **키드 매칭 배포**(② 로직) + **급변 알림**(③). **Claude(web)+Codex(배포)**.
4. **주1회 스캔 자동화** 가동. **Codex**.
> 순서 중요: DB 정리(1) → 시트 재생성(2) → 키드 매칭 켜기(3). 순서 어기면 다시 섞임.

## 정직한 잔여 리스크
- ①②③으로 **구조적 번짐은 멈춤**. 단 **사람이 시트에 새로 넣는 오타·드래그 실수**는 양방향인 한 완전차단 불가 → 복사-가드+주간 스캔으로 **조기 감지·차단**(alert-only, 사람이 정정).
- 완전 원천봉쇄가 필요하면 아래 단방향.

## (참고) 더 강한 대안: 단방향 DB 단일정본
importStats(시트→DB) 폐지, 수기입력=대시보드로 일원화, 시트=export 표시전용. 시트로 인한 오염 **원천 불가**. 단 팀 워크플로가 "시트 입력→대시보드 입력"으로 바뀜. 이번엔 미채택(사용자 결정).
