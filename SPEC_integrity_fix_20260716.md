## #1 — "필터가 달라 값이 숨는" 문제 제거

**증상:** `measured_at`이 `ended_at` 이후인 값을 `stats-for-sheet`/exportStats가 **조용히 제외** → 시트엔 빈칸인데 대시보드엔 보임. (하토토 98,362@7/13 사례. 원인은 URL 아님, **날짜였음**)

**요구 동작:**
1. **입력 검증(쓰기 시점):** 종료된 게시물에 `measured_at > ended_at`인 값이 저장되지 않게 한다.
   - 적용 경로 **전부**: `stats-import`(시트→DB), 대시보드 PATCH, syncAll/bulk, run_monitoring.
   - 처리: **차단+경고**(권장) 또는 **ended_at로 클램프**(팀 관례=최종값은 종료일에 귀속) 중 택1하되, 화면/시트 **양쪽에서 동일하게** 보이도록.
   - ⚠️ 종료 게시물은 자동수집이 이미 스킵하므로, 이 값은 사실상 수동/정리 유입 → 관례상 **종료일 귀속**이 자연스러움.
2. **필터 로직 단일화:** 시트로 내보내는 규칙(`stats-for-sheet`)과 대시보드 표시 규칙을 **하나의 공유 함수**로 통일. "한쪽엔 보이고 한쪽엔 안 보임"이 원천적으로 못 생기게.
3. 회귀 테스트: 종료일 이후 값이 들어와도 시트·대시보드가 **같은 값**을 보이는지.

## #2 — "같은 영상이 2개로 중복 등록" 문제 제거

**증상:** 같은 영상이 URL 변형(www 유무, `/reel`↔`/p`)으로 **별도 게시물 2개**로 등록 → 이중 계상. (류라이 틱톡 `7652295124399000839` 사례)

**현재 상태:** 기존 중복은 **이미 정리됨(DB 중복 0건, Claude 확인 2026-07-16).** 아래는 **재발 방지 잠금**.

**요구 동작:**
1. **정규화 키 정의(서버 공통 함수):**
   - IG = shortcode (`/p`·`/reel`·`/reels`·`/tv` 통일)
   - 틱톡 = 영상ID (`/video/\d+`, www 유무 무시)
   - 유튜브 = 영상ID (`/shorts/…`)
   - (Apps Script `linkKey_`, 서버 정규화와 **같은 기준**)
2. **DB 고유 제약:** `sponsored_posts`에 정규화 키 컬럼(예: `link_key`) 추가 + **UNIQUE 인덱스**. 또는 등록 시 정규화 키로 먼저 조회 → 있으면 **update, 없을 때만 insert**.
3. **URL 표준형 저장:** 저장 시 표준형으로 정규화(IG→`/p/`, 틱톡→**www 포함**). ⚠️ 현재 살아있는 류라이 레코드 `e32284d3` URL이 `tiktok.com`(www 없음) → **www 형으로 교정** 필요(수집 actor는 www 필수).
4. 마이그레이션: 기존 962행 URL 정규화 백필 + `link_key` 채우기(중복 재발생 여부 사전 점검).
5. 회귀 테스트: www/no-www·`/reel`/`/p` 다른 URL을 등록해도 **1개로 합쳐지는지**.

## #3 — 충돌 해소 규칙 (사용자 확정)

**규칙: "마지막으로 수동 수정된 값이 최종."** 시트든 대시보드든 **나중에 손댄 수동 값이 이김.**
- 기존 정책 `stats-input-latest-wins`(커밋 3852fff)와 동일 방향 → **모든 쓰기 경로에 일관 적용** 확인만.
- 단, mono(하향 방지) 가드가 **수동 하향 정정을 되돌리지 않도록** 예외 유지(커밋 2722cf4 취지).
- "시트=유일 정답(B안)"은 **채택 안 함** — 자동수집 일별값이 시트엔 없을 수 있어 시트 빈칸이 DB를 지우면 안 되기 때문.
